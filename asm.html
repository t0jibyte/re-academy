01 — What is Assembly?

Computers only understand 1s and 0s (binary machine code). Assembly language is just a human-readable text version of those 1s and 0s.

Instead of typing 10111000 00000101 00000000 00000000 00000000, you type mov eax, 5. They mean exactly the same thing to the processor. When you compile a C program, the compiler translates your C code into Assembly, and then an assembler translates that into binary.

02 — CPU Registers

If you want to do math quickly, you can't rely on RAM because it's too slow. Instead, the CPU has tiny, ultra-fast storage boxes actually built into the silicon. These are called registers.

You can think of registers like built-in variables. On a 64-bit CPU, they hold 64 bits (8 bytes) of data. Some of the most common general-purpose registers are:

  • RAX: Often used to hold the result of a function.
  • RBX, RCX, RDX: Used for general data and calculations.
  • RDI, RSI: Often used to hold arguments being passed to functions.
  • RIP: The Instruction Pointer. It points to the next instruction the CPU should run.

03 — The mov Instruction

The most common instruction you will ever see is mov (short for "move"). It moves data from one place to another. Actually, "copy" is a better word, because the original data doesn't disappear.

The syntax we use (Intel syntax) is always mov DESTINATION, SOURCE.

using mov; Put the number 5 into the RAX register
mov  rax, 5

; Copy the value of RAX into RBX (RBX is now 5)
mov  rbx, rax

; To work with memory, use square brackets [].
; This means "go to the memory address inside RCX, and read the data there into RAX"
mov  rax, [rcx]

04 — Basic Math

Once you have data in registers, you can do math on it using instructions like add and sub. 

arithmetic instructionsmov  rax, 10     ; rax = 10
add  rax, 5      ; rax = rax + 5  (now 15)
sub  rax, 2      ; rax = rax - 2  (now 13)

; You can also add another register:
mov  rbx, 3
add  rax, rbx     ; rax = 13 + 3 = 16

05 — The Stack

The CPU has very few registers. If you run out of them, you can store data temporarily in RAM using a structure called the Stack.

Think of it like a stack of plates. You can push a new plate onto the top of the stack (push), or take the top plate off (pop).

push and popmov  rax, 99
push rax      ; saves 99 onto the top of the stack
mov  rax, 0       ; rax is now overwritten with 0

; Later, when we want our 99 back:
pop  rbx      ; removes top value from stack and places it in rbx
; rbx is now 99!

06 — Syscalls (Hello World)

Assembly doesn't have printf() or "print to screen" commands. If you want to print something or read a file, you have to ask the Operating System (Linux) to do it for you using a syscall.

You do this by putting a specific "Phone Number" in the RAX register to tell Linux what service you want, putting the arguments in other registers, and then yelling syscall.

Syscall: write(1, "HI\\n", 3)mov  rax, 1          ; System call ID 1 is "sys_write"
mov  rdi, 1          ; File Descriptor 1 is "stdout" (the terminal screen)
lea  rsi, [rip+msg]  ; Memory address of our string
mov  rdx, 3          ; Length of the string ("HI\n" is 3 bytes)
syscall              ; Ask Linux to run it!

; Wait, how do we exit safely? Syscall 60!
mov  rax, 60         ; System call ID 60 is "sys_exit"
mov  rdi, 0          ; Exit code 0 (success)
syscall
