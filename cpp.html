C++ From Zero

C++ is one of the most powerful and widely used languages in the world. It gives you high-level features like classes and vectors, but also gives you low-level access to memory. Here, we will learn C++ absolutely from scratch, step by step.

01 — Hello C++

Every C++ program starts with the main() function. To print text to the screen, we use std::cout (character output) and the << operator. We must also include the <iostream> library at the top of our file to tell the compiler what cout is. 

hello.cpp#include <iostream>

int main() {
    // Print text to the console, followed by a new line (endl)
    std::cout << "Hello, World!" << std::endl;
    
    // Return 0 means the program ran successfully
    return 0;
}
// tipYou will often see using namespace std; which allows you to type cout instead of std::cout. However, typing std:: is considered best practice to prevent naming conflicts!

02 — Variables and Data Types

C++ is statically typed, which means you must declare the type of a variable before you use it, and you cannot change its type later.

Type Description Example
int Whole numbers (integers) int age = 25;
double Decimal numbers (floating point) double price = 19.99;
char Single character (use single quotes) char grade = 'A';
bool True or false values bool isGameOver = false;
std::string Text (use double quotes, needs <string>) std::string name = "Alice"; 
variables.cpp#include <iostream>
#include <string>

int main() {
    int health = 100;
    std::string playerName = "Hero";
    
    // You can chain outputs together!
    std::cout << playerName << " has " << health << " HP." << std::endl;
    return 0;
}

03 — Control Flow (If / Else)

C++ uses if, else if, and else to make decisions. The conditions must be inside parentheses (), and the code to run goes inside curly braces {}.

logic.cppint score = 85;

if (score >= 90) {
    std::cout << "Grade: A" << std::endl;
} else if (score >= 80) {
    std::cout << "Grade: B" << std::endl;
} else {
    std::cout << "Grade: C or lower" << std::endl;
}

// Logical Operators:
// &&  (AND) - true if BOTH are true
// ||  (OR)  - true if at least ONE is true
// !   (NOT) - flips true to false
if (score > 0 && score <= 100) {
    std::cout << "Valid score." << std::endl;
}

04 — Loops (For and While)

Loops allow you to run the same block of code multiple times. A for loop is used when you know exactly how many times you want to loop. A while loop runs as long as a condition is true.

loops.cpp// 1. The FOR loop
// Structure: for (initialization; condition; update)
for (int i = 0; i < 5; i++) {
    std::cout << "Count: " << i << std::endl;
}
// Output: 0, 1, 2, 3, 4

// 2. The WHILE loop
int hp = 3;
while (hp > 0) {
    std::cout << "Taking damage! HP left: " << hp << std::endl;
    hp--; // decrease hp by 1
}

03 — Constructors, Destructors, Copy, Move

C++class Buffer {
    uint8_t *data;
    size_t   size;

public:
    // Constructor — initializer list is faster than assignment in body
    Buffer(size_t n) : size(n), data(new uint8_t[n]) {
        memset(data, 0, n);
    }

    // Destructor — runs when object dies (scope exit, delete, exception)
    ~Buffer() {
        delete[] data;   // MUST use delete[] for arrays
        data = nullptr;
    }

    // Copy constructor — deep copy
    Buffer(const Buffer& other) : size(other.size), data(new uint8_t[other.size]) {
        memcpy(data, other.data, size);
    }

    // Move constructor — steal resources, leave other empty
    Buffer(Buffer&& other) : size(other.size), data(other.data) {
        other.data = nullptr;
        other.size = 0;
    }

    // If you define any of these, define all five (rule of five):
    // constructor, destructor, copy ctor, copy assign, move ctor/assign
};

04 — Inheritance & Polymorphism

C++class Animal {
public:
    std::string name;
    Animal(const std::string& n) : name(n) {}
    virtual void speak() { std::cout << "..."; }
    virtual ~Animal() {}   // virtual destructor: REQUIRED for polymorphism
                            // without it: deleting Dog through Animal* leaks memory
};

class Dog : public Animal {
public:
    Dog(const std::string& n) : Animal(n) {}
    void speak() override {   // override: compiler checks it actually overrides
        std::cout << name << ": Woof!\n";
    }
};

// Polymorphism: Animal* can point to Dog, Cat, etc.
// The right speak() is chosen at RUNTIME via vtable
void make_speak(Animal *a) {
    a->speak();   // which speak()? decided at runtime by vtable lookup
}

Dog d("Rex");
make_speak(&d);   // calls Dog::speak — "Rex: Woof!"

// Abstract class: cannot be instantiated, forces subclasses to implement
class Shape {
public:
    virtual double area() const = 0;   // pure virtual — = 0 means no implementation
    virtual ~Shape() {}
};
// Shape s;    // error: cannot instantiate abstract class

05 — How Virtual Dispatch Actually Works

When a class has any virtual method, the compiler adds a hidden vptr (virtual pointer) at offset 0 of the object. This pointer points to the class's vtable — a read-only array of function pointers in .rodata. A virtual call is just: load vptr → index into vtable → call. 

virtual dispatch — step by step// C++ source:
Animal *a = new Dog("Rex");
a->speak();

// What the compiler generates (x64 assembly):
mov  rdi, a          ; rdi = "this" pointer (ptr to Dog object)
mov  rax, [rdi]      ; load vptr from offset 0 of object
mov  rax, [rax+8]    ; load vtable[1] (index 1 = speak, 0 = destructor)
call rax             ; call through pointer

// Dog object in memory:
// [vptr    8B] → Dog's vtable in .rodata
// [name  32B] → std::string (contains "Rex")

// Dog's vtable in .rodata:
// vtable_Dog:                 
//   [0] → address of Dog::~Dog()     
//   [8] → address of Dog::speak()    

// Exploit: overwrite vptr → redirect ALL virtual calls on that object

06 — vtable Memory Layout — In Detail

vtable layout in memory// Class with 2 virtual methods:
class Base {
public:
    virtual void foo();   // vtable[0]
    virtual void bar();   // vtable[1]
    virtual ~Base();      // vtable[2] and vtable[3] (deleting dtor)
    int x;
};

// Base object layout (24 bytes):
// offset 0: vptr  (8 bytes) → &Base::vtable
// offset 8: x     (4 bytes)
// offset 12: pad  (4 bytes)

// Base vtable in .rodata:
// [typeinfo pointer]     (used by RTTI/dynamic_cast)
// [Base::~Base dtor]
// [Base::foo]
// [Base::bar]

// Derived overrides foo:
class Derived : public Base {
public:
    void foo() override;   // new vtable[1] = &Derived::foo
    int y;
};

// Derived vtable in .rodata:
// [typeinfo pointer]     (Derived's typeinfo, different from Base's)
// [Derived::~Derived]
// [Derived::foo]         ← replaced with Derived's version
// [Base::bar]            ← inherited — still points to Base::bar

// Read vtable manually in GDB (pwndbg):
// info vtbl obj          — show vtable of object
// x/4gx *(void**)&obj    — read 4 qwords from vtable

// Read vtable in C++ code:
Derived d;
void** vptr = *(void***)&d;
printf("vtable @ %p\n", vptr);
for (int i = 0; i < 4; i++)
    printf("  [%d] = %p\n", i, vptr[i]);
// vtable corruption — a real exploit primitiveIf you can

05 — Functions

Functions allow you to organize code into reusable blocks. They must declare what type of data they return (or void if they return nothing), and what parameters they accept.

functions.cpp#include <iostream>

// Function definition: returns an int, takes two ints as input
int addNumbers(int a, int b) {
    return a + b;
}

// A void function doesn't return anything
void greet(std::string name) {
    std::cout << "Hello " << name << "!" << std::endl;
}

int main() {
    int sum = addNumbers(5, 10);
    std::cout << "5 + 10 = " << sum << std::endl;
    
    greet("Alice");
    return 0;
}

06 — Vectors (Smart Arrays)

Unlike raw C arrays, a std::vector can grow and shrink in size automatically. It is part of the C++ Standard Template Library (STL). You must #include <vector> to use it. 

vectors.cpp#include <iostream>
#include <vector>

int main() {
    // Create a vector of integers
    std::vector<int> scores = {90, 85, 100};
    
    // Add a new element to the end
    scores.push_back(95);
    
    // Loop through the vector
    std::cout << "All scores: ";
    for (int i = 0; i < scores.size(); i++) {
        std::cout << scores[i] << " ";
    }
    std::cout << std::endl;
    
    // Range-based for loop (easier way to loop!)
    for (int score : scores) {
        // Do something with score
    }
    
    return 0;
}

07 — Pointers & References

When you pass a variable to a function, C++ makes a copy of it. If you want a function to modify the original variable, you must pass it by reference using &. 

references.cpp// This function takes 'a' by reference. It modifies the original!
void levelUp(int& playerLevel) {
    playerLevel++;
}

int main() {
    int myLevel = 5;
    std::cout << "Before: " << myLevel << std::endl;
    
    levelUp(myLevel);  // Passes the actual variable, not a copy
    
    std::cout << "After: " << myLevel << std::endl; // prints 6
    return 0;
}
// what about pointers?Pointers (int* ptr = &myLevel;) are the older C-style way to share memory. In modern C++, References (int& ref = myLevel;) are vastly preferred because they are safer and cannot be null.

08 — Structs & Classes

Classes allow you to bundle data (variables) and behavior (functions) together into Objects. This is the core of Object-Oriented Programming (OOP).

oop.cppclass Player {
private:
    // Private data: cannot be accessed directly from main()
    std::string name;
    int health;

public:
    // Constructor: called automatically when object is created
    Player(std::string n, int h) {
        name = n;
        health = h;
    }
    
    // Methods (functions inside a class)
    void takeDamage(int dmg) {
        health -= dmg;
        std::cout << name << " took " << dmg << " damage!" << std::endl;
    }
};

int main() {
    // Create an object of type Player
    Player p1("Hero", 100);
    p1.takeDamage(25); // call the object's method
    return 0;
}
